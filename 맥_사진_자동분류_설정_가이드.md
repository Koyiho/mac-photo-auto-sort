# 맥 사진 자동 분류 설정 가이드

`PhotoInbox` 폴더에 사진/동영상을 넣으면 촬영일(EXIF) 기준으로
`PhotoSorted/연도/월/일` 폴더에 자동으로 정리해주는 시스템입니다.
macOS 기본 기능(Folder Action)과 무료 오픈소스 도구(exiftool)만 사용합니다.

---

## 완성된 구조

- 감시 폴더: `~/Pictures/PhotoInbox` (여기에 사진을 넣으면 자동 분류됨)
- 결과 폴더: `~/Pictures/PhotoSorted/2025/06/22/` 형식으로 자동 생성
- 스크립트 위치: `~/Scripts/organize_photos.sh`
- 로그 파일: `~/Library/Logs/photo_organizer.log`

---

## 1단계: Homebrew 설치 (이미 있으면 생략)

터미널(Terminal.app)을 열고:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

## 2단계: exiftool 설치

```bash
brew install exiftool
```

설치 확인:

```bash
which exiftool
```

`/opt/homebrew/bin/exiftool` 같은 경로가 나오면 정상입니다.

## 3단계: 폴더 구성

Finder에서:

1. `Cmd + Shift + H` → 홈 폴더 이동
2. 새로운 폴더 생성 → 이름 `Scripts`
3. `그림(Pictures)` 폴더 안에 새로운 폴더 생성 → 이름 `PhotoInbox`

## 4단계: 스크립트 저장

아래 스크립트를 `organize_photos.sh` 라는 이름으로 저장한 뒤,
`~/Scripts/` 폴더 안에 넣습니다.

```bash
#!/bin/bash
#
# organize_photos.sh
# ------------------------------------------------------------
# 감시 폴더에 새로 들어온 사진/동영상을 촬영일(EXIF DateTimeOriginal)
# 기준으로 ~/Pictures/PhotoSorted/연도/월/일 폴더에 자동으로 옮겨줍니다.
#
# 사용법:
#   ./organize_photos.sh 파일1 파일2 ...
#   (macOS Folder Action이 새로 추가된 파일들을 인자로 자동 전달해줍니다)
#
# 사전 준비:
#   brew install exiftool
# ------------------------------------------------------------

# Automator/Folder Action은 터미널과 다른 PATH를 사용하기 때문에
# Homebrew로 설치한 exiftool 위치를 명시적으로 PATH에 추가해줍니다.
export PATH="/opt/homebrew/bin:/usr/local/bin:$PATH"

# 사진을 정리해서 모아둘 최상위 폴더 (원하면 경로 수정 가능)
DEST_ROOT="$HOME/Pictures/PhotoSorted"

# 처리 기록을 남길 로그 파일
LOG_FILE="$HOME/Library/Logs/photo_organizer.log"

mkdir -p "$DEST_ROOT"
mkdir -p "$(dirname "$LOG_FILE")"

log() {
  echo "$(date '+%Y-%m-%d %H:%M:%S') $1" >> "$LOG_FILE"
}

for f in "$@"; do
  # 폴더는 건너뜀
  [ -d "$f" ] && continue
  # 파일이 실제 존재하는지 확인 (이동/삭제 중일 수 있음)
  [ -f "$f" ] || continue

  # 처리할 확장자만 필터링 (필요하면 목록에 추가/삭제 가능)
  ext="${f##*.}"
  ext_lower=$(echo "$ext" | tr '[:upper:]' '[:lower:]')
  case "$ext_lower" in
    jpg|jpeg|png|heic|heif|tiff|tif|raw|cr2|cr3|nef|arw|dng|mov|mp4|gif|bmp|webp) ;;
    *) continue ;;
  esac

  # exiftool 설치 여부 확인
  if ! command -v exiftool >/dev/null 2>&1; then
    log "오류: exiftool이 설치되어 있지 않습니다. 'brew install exiftool'로 설치하세요."
    exit 1
  fi

  # 1순위: 촬영일(DateTimeOriginal), 2순위: 생성일(CreateDate), 3순위: 파일 자체의 수정일
  date_str=$(exiftool -DateTimeOriginal -d "%Y-%m-%d" -s -s -s "$f" 2>/dev/null)
  if [ -z "$date_str" ]; then
    date_str=$(exiftool -CreateDate -d "%Y-%m-%d" -s -s -s "$f" 2>/dev/null)
  fi
  if [ -z "$date_str" ]; then
    date_str=$(stat -f "%Sm" -t "%Y-%m-%d" "$f")
  fi

  year="${date_str%%-*}"
  rest="${date_str#*-}"
  month="${rest%%-*}"
  day="${rest##*-}"

  # 혹시 연/월/일 파싱이 이상하면 "미분류" 폴더로
  if [ -z "$year" ] || [ -z "$month" ] || [ -z "$day" ] || [ "$month" = "$day" ]; then
    dest_dir="$DEST_ROOT/미분류"
  else
    dest_dir="$DEST_ROOT/$year/$month/$day"
  fi
  mkdir -p "$dest_dir"

  base=$(basename "$f")
  target="$dest_dir/$base"

  # 같은 이름 파일이 이미 있으면 번호를 붙여서 저장
  if [ -e "$target" ]; then
    name="${base%.*}"
    extf="${base##*.}"
    i=1
    while [ -e "$dest_dir/${name}_$i.$extf" ]; do
      i=$((i+1))
    done
    target="$dest_dir/${name}_$i.$extf"
  fi

  mv "$f" "$target"
  log "이동됨: $f -> $target"
done
```

실행권한 부여:

```bash
chmod +x ~/Scripts/organize_photos.sh
```

## 5단계: macOS Folder Action 연결

1. `Automator` 앱 실행
2. 새로운 문서 → **폴더 적용 스크립트(Folder Action)** 선택
3. 상단 "폴더 적용 스크립트는 다음 폴더에 추가되는 파일 및 폴더를 받습니다" 드롭다운에서
   `PhotoInbox` 폴더 선택 (없으면 "기타..."로 찾아서 지정)
4. 왼쪽 카테고리에서 **유틸리티** 선택 → **셸 스크립트 실행** 항목을
   오른쪽 빈 워크플로우 영역으로 드래그
5. 옵션 설정:
   - **셸**: `/bin/bash` (또는 `/bin/zsh`도 무방 — 스크립트 자체에 `#!/bin/bash`가 있어 자동 처리됨)
   - **통과 입력**: `인수` (as arguments)
   - 스크립트 입력창에 아래 한 줄만 입력:
     ```
     ~/Scripts/organize_photos.sh "$@"
     ```
6. `Cmd + S` 저장 → 이름 `PhotoAutoSort`
7. 저장하면 자동으로 `PhotoInbox` 폴더에 동작이 연결됩니다.

## 6단계: 테스트

`PhotoInbox` 폴더에 사진 몇 장을 드래그해서 넣고 몇 초 기다린 뒤:

```bash
ls -R ~/Pictures/PhotoSorted
```

`2025/06/22` 같은 폴더 안에 사진이 들어가 있으면 성공입니다.

## 트러블슈팅

### 사진을 넣어도 아무 반응이 없을 때

로그 확인:

```bash
cat ~/Library/Logs/photo_organizer.log
```

**"exiftool이 설치되어 있지 않습니다" 오류가 뜨는 경우**
`which exiftool`로는 잘 나오는데 로그에는 없다고 뜬다면,
Automator(Folder Action)가 터미널과 다른 PATH 환경에서 실행되기 때문입니다.
이 가이드의 스크립트에는 이미 아래 줄로 해결되어 있습니다:

```bash
export PATH="/opt/homebrew/bin:/usr/local/bin:$PATH"
```

**Folder Action은 "새로 추가된" 파일에만 반응합니다**
이미 폴더 안에 있던 파일은 자동으로 처리되지 않습니다.
기존 파일을 수동으로 정리하려면:

```bash
~/Scripts/organize_photos.sh ~/Pictures/PhotoInbox/*
```

**이미 분류된 사진들을 다시 정리하고 싶을 때** (예: 연/월만 있던 걸 연/월/일로 재정리)

```bash
find ~/Pictures/PhotoSorted -maxdepth 3 -type f \
  \( -iname "*.jpg" -o -iname "*.jpeg" -o -iname "*.png" -o -iname "*.heic" -o -iname "*.mov" -o -iname "*.mp4" \) \
  -print0 | xargs -0 ~/Scripts/organize_photos.sh
```

### Automator에서 헷갈렸던 부분들 (참고용)

- 새 문서 만들 때 "응용 프로그램"이 아니라 반드시 **"폴더 적용 스크립트(Folder Action)"** 를 선택해야 함
- "동작" 탭과 "변수" 탭을 헷갈리기 쉬움 — 반드시 **"동작"** 탭에서 액션을 찾아야 함
- "셸 스크립트"라는 이름의 **변수**와 "셸 스크립트 실행"이라는 **동작**은 다른 것 — 동작 쪽을 사용해야 함
- "셸 스크립트 실행"은 **유틸리티** 카테고리 안에 있음

## 나중에 바꾸고 싶을 때

- **분류 기준 변경** (예: 카메라 기종별 추가): `organize_photos.sh` 안의
  `dest_dir="$DEST_ROOT/$year/$month/$day"` 부분을 수정하면 됩니다.
- **감시 폴더 변경**: Automator에서 저장한 `PhotoAutoSort` 파일을 다시 열어
  상단 드롭다운에서 다른 폴더로 바꾸면 됩니다.
- **분류 결과 저장 위치 변경**: 스크립트 안의 `DEST_ROOT` 값을 수정하면 됩니다.
